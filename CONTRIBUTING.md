## How to Contribute

### Reporting Bugs
- Use GitHub Issues
- Include detailed description of the bug
- Provide steps to reproduce
- Include error messages and logs

### Suggesting Enhancements
- Open a GitHub Issue with the "enhancement" label
- Describe the feature and its use case
- Explain why it would be valuable

### Pull Requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Test thoroughly
5. Commit your changes (`git commit -m 'Add amazing feature'`)
6. Push to the branch (`git push origin feature/amazing-feature`)
7. Open a Pull Request

### Code Standards
- Follow Python PEP 8 style guide
- Use type hints
- Add docstrings to functions
- Test your changes
- Follow CodeWords best practices

## Development Setup

See [README.md](README.md) for local development setup instructions.

## Code Review Process

1. All PRs require at least one review
2. Tests must pass
3. Code must follow style guidelines
4. Documentation must be updated

Feel free to open an issue or reach out via the CodeWords platform!
